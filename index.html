python内置函数分析(1)

PYTHON中内置了一些巧妙而强大的内置函数，熟练使用这些函数，代码不仅简洁易读了许多，既方便有减少了很多的BUG。

1.sorted()

1)对一个列表进行排序

a = [100,82,12,55,45]
sorted(a)
>>>[12,45,55,88,100]

2)通过key参数/函数

例：长列表中嵌套字典元素,按照每个元素的长度进行排序(这里使用匿名函数lambda)

list1 = [{2:6,8:2},{1:7},{1:2,5:12,9:2}]
new_list = sorted(list1,key = lambda x:len(x))
print(new_list)
>>>[{1: 7}, {8: 2, 2: 6}, {1: 2, 5: 12, 9: 2}]

3)对由tuple组成的list进行排序

例：对一群人里面的年龄的list进行排序

people = [('laownag','man','23'),('xiaoli','woman','21'),('xiaozhang','man','17')]
new_people = sorted(people,key = lambda x:x[2])
print(new_people)
>>>[('xiaozhang', 'man', '17'), ('xiaoli', 'woman', '21'), ('laownag', 'man', '23')]

4)用cmp函数排序

python3中，cmp函数被operator模块取代，需要导入
cmp(x,y),如果x>y,返回1，x<y,返回-1，x==y,返回0

people = [('laownag','man','23'),('xiaoli','woman','21'),('xiaozhang','man','17')]
new_people = sorted(people,cmp = lambda x，y:cmp(x[0],y[0])))
print(new_people)
>>>[('laownag', 'man', '23'), ('xiaoli', 'woman', '21'), ('xiaozhang', 'man', '17')]

2.map()

map函数跟根据提供的函数对制定序列做排序，接受一个函数f和一个list，并把函数f以此作用于每个list函数，然后返回一个新的list,map的函数的参数可以为多个，注意:这个函数一定要有返回值

def func(x):
    return x*x
print(map(func([1,2,3,4,5])))
>>>[1, 4, 9, 16, 25]

def add_x_y(x,y):
return x+y
print (map(add_x_y,[1,2,3,4,5],[2,3,4,5,6]))#必须len(x)==len(y)
>>>[3, 5, 7, 9, 11]

4.zip()

zip函数接收人一多个(包括0或1)序列作为参数，返回一个tuple

示例1

x=[1,2,3]
y=[4,5,6]
z=[7,8,9]
xyz=zip(x,y,z)
print(xyz)
>>>[(1, 4, 7), (2, 5, 8), (3, 6, 9)]

示例2

x = [1, 2, 3]
y = [4, 5, 6, 7]
xy = zip(x, y)
print xy
>>>[(1, 4), (2, 5), (3, 6)]

示例3

#示例3
x = [1, 2, 3]
y = [4, 5, 6]
z = [7, 8, 9]
xyz = zip(x, y, z)
u = zip(*xyz)
print u
>>>[(1, 2, 3), (4, 5, 6), (7, 8, 9)]

一般认为这是一个unzip的过程，它的运行机制是这样的：
在运行zip(*xyz)之前，xyz的值是：[(1, 4, 7), (2, 5, 8), (3, 6, 9)]
那么，zip(*xyz) 等价于 zip((1, 4, 7), (2, 5, 8), (3, 6, 9))
所以，运行结果是：[(1, 2, 3), (4, 5, 6), (7, 8, 9)]

#示例4
x = [1, 2, 3]
r = zip(* [x] * 3)
print r
>>>[(1, 1, 1), (2, 2, 2), (3, 3, 3)]

>
它的运行机制是这样的：
[x]生成一个列表的列表，它只有一个元素x
[x] * 3生成一个列表的列表，它有3个元素，[x, x, x]
zip(* [x] * 3)的意思就明确了，zip(x, x, x)

5.filter()

filter函数是接收一函数f,和一个list，这个函数是对每个函数进行判断返回True或者False,依次来过滤掉一些不符合条件的元素，返回符合元素的list.跟if判断添加list相似

def is_num(x):
    return x%2==0
print(filter(is_num,[1,2,3,4,5,6,7]))
>>>[2, 4, 6]

处理文件时，把一些不需要的空格，回车，空字符去掉

Veröffentlicht am 2017-04-01

1.算法概念

<1>算法是计算处理信息的本质

  • 计算机程序本质上是一个算法来告诉计算机确切的步骤来执行一个指定的任务。
  • 算法是独立存在的一种解决问题的思想和算法。

<2>算法的五大特征:

  1. 输入：算法具有零个或多个输入
  2. 输出:算法至少具有一个或多个输出
  3. 有穷性: 算法在有限的步骤之后会自动结束而不会无限循环，并且每一个步骤可以在可接受的时间内完成
  4. 确定性：算法中的每一步都有确定的含义，不会出现二义性
  5. 可行性：算法的每一步都是可行的，也就是说每一步都能够执行有限的次数完成

<3>算法效率衡量

  1. 执行时间反应算法效率：实现算法程序的执行时间可以反应出算法的效率，即算法的优劣。
  2. 单纯依靠运行的时间来比较算法的优劣并不一定是客观准确：程序的运行离不开计算机环境（包括硬件和操作系统），这些客观原因会影响程序运行的速度并反应在程序的执行时间上。
  3. 时间复杂度：假设存在函数g，使得算法A处理规模为n的问题示例所用时间为T(n)=O(g(n))，则称O(g(n))为算法A的渐近时间复杂度，简称时间复杂度，记为T(n)
  4. 最坏时间复杂度:
    • 算法完成工作最少需要多少基本操作，即最优时间复杂度
    • 算法完成工作最多需要多少基本操作，即最坏时间复杂度
    • 算法完成工作平均需要多少基本操作，即平均时间复杂度
  5. 最优时间复杂度，反映的只是最乐观最理想的情况，没有参考价值。
  6. 最坏时间复杂度表明算法在此种程度的基本操作中一定能完成工作。
  7. 平均时间复杂度是对算法的一个全面评价它完整全面的反映了这个算法的性质。但这种衡量并没有保证，不是每个计算都能在这个基本操作内完成，也会因为应用算法的实例分布可能并不均匀而难以计算。

因此，我们主要关注算法的最坏情况，亦即最坏时间复杂度。

时间复杂度的几条基本计算规则:

  1. 基本操作，即只有常数项，认为其时间复杂度为O(1)
  2. 顺序结构，时间复杂度按加法进行计算
  3. 循环结构，时间复杂度按乘法进行计算
  4. 分支结构，时间复杂度取最大值
  5. 判断一个算法的效率时，往往只需要关注操作数量的最高次项，其它次要项和常数项可以忽略
  6. 在没有特殊说明时，我们所分析的算法的时间复杂度都是指最坏时间复杂度

<6>数据结构

1.概念：

数据是一个抽象的概念，将其进行分类后得到程序设计语言中的基本类型。如：int，float，char等。数据元素之间不是独立的，存在特定的关系，这些关系便是结构。数据结构指数据对象中数据元素之间的关系。

2.算法与数据结构的区别

数据结构只是静态的描述了数据元素之间的关系。

高效的程序需要在数据结构的基础上设计和选择算法。

程序 = 数据结构 + 算法

总结：算法是为了解决实际问题而设计的，数据结构是算法需要处理的问题载体

正则表达式

正则表达式：(Regular Expression)是计算机科学的一个概念。Regular Expression即“描述某种规则的表达式”之意。

正则表达式使用单个字符串来描述、匹配一系列匹配某个句法规则的字符串。在很多文本编辑器里，正则表达式通常被用来检索、替换那些匹配某个模式的文本。

1. 在Python中需要通过正则表达式对字符串进行匹配的时候通常引入：re模块

re.match是用来进行正则匹配检查的方法，若字符串匹配正则表达式，则match方法返回匹配对象（Match Object），否则返回None（注意不是空字符串””）。

匹配对象Macth Object具有group方法，用来返回字符串的匹配部分。

2. re模块示例(匹配以itcast开头的语句)

#coding=utf-8

import re

result = re.match("itcast","itcast.cn")

result.group()

运行结果为：

itcast

6.正则表达式中表示数量

字符| 功能

*|匹配前一个字符出现0次或者无限次，即可有可无

+|匹配前一个字符出现1次或者无限次，即至少有1次

?|匹配前一个字符出现1次或者0次，即要么有1次，要么没有

{m} |匹配前一个字符出现m次

{m,} |匹配前一个字符至少出现m次

{m,n} |匹配前一个字符出现从m到n次

10.贪婪和非贪婪

Python里数量词默认是贪婪的（在少数语言里也可能是默认非贪婪），总是尝试匹配尽可能多的字符；

非贪婪则相反，总是尝试匹配尽可能少的字符。

在”*”,”?”,”+”,”{m,n}”后面加上？，使贪婪变成非贪婪。

>>> s="This is a number 234-235-22-423"
>>> r=re.match(".+(\d+-\d+-\d+-\d+)",s)
>>> r.group(1)
'4-235-22-423'
>>> r=re.match(".+?(\d+-\d+-\d+-\d+)",s)
>>> r.group(1)
'234-235-22-423'
>>>

正则表达式模式中使用到通配字，那它在从左到右的顺序求值时，会尽量“抓取”满足匹配最长字符串，在上面的例子里面，“.+”会从字符串的启始处抓取满足模式的最长字符，其中包括想得到的第一个整型字段的中的大部分，“\d+”只需一位字符就可以匹配，所以它匹配了数字“4”，而“.+”则匹配了从字符串起始到这个第一位数字4之前的所有字符。

>>> re.match(r"aa(\d+)","aa2343ddd").group(1)
'2343'
>>> re.match(r"aa(\d+?)","aa2343ddd").group(1)
'2'
>>> re.match(r"aa(\d+)ddd","aa2343ddd").group(1) 
'2343'
>>> re.match(r"aa(\d+?)ddd","aa2343ddd").group(1)
'2343'
>>>

以上就是关于正则表达式基本知识，几乎所有的正则表达式都可以通过以上方法就行匹配，正则表达式广泛应用于检索，替换那些符合某个模式(规则)的文本，在python的爬虫应用中可以用来检索爬取检索到的有用信息。
