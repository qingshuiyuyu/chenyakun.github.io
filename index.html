Python-数据结构与算法简单入门

1.算法概念

<1>算法是计算处理信息的本质

  • 计算机程序本质上是一个算法来告诉计算机确切的步骤来执行一个指定的任务。
  • 算法是独立存在的一种解决问题的思想和算法。

<2>算法的五大特征:

  1. 输入：算法具有零个或多个输入
  2. 输出:算法至少具有一个或多个输出
  3. 有穷性: 算法在有限的步骤之后会自动结束而不会无限循环，并且每一个步骤可以在可接受的时间内完成
  4. 确定性：算法中的每一步都有确定的含义，不会出现二义性
  5. 可行性：算法的每一步都是可行的，也就是说每一步都能够执行有限的次数完成

<3>算法效率衡量

  1. 执行时间反应算法效率：实现算法程序的执行时间可以反应出算法的效率，即算法的优劣。
  2. 单纯依靠运行的时间来比较算法的优劣并不一定是客观准确：程序的运行离不开计算机环境（包括硬件和操作系统），这些客观原因会影响程序运行的速度并反应在程序的执行时间上。
  3. 时间复杂度：假设存在函数g，使得算法A处理规模为n的问题示例所用时间为T(n)=O(g(n))，则称O(g(n))为算法A的渐近时间复杂度，简称时间复杂度，记为T(n)
  4. 最坏时间复杂度:
    • 算法完成工作最少需要多少基本操作，即最优时间复杂度
    • 算法完成工作最多需要多少基本操作，即最坏时间复杂度
    • 算法完成工作平均需要多少基本操作，即平均时间复杂度
  5. 最优时间复杂度，反映的只是最乐观最理想的情况，没有参考价值。
  6. 最坏时间复杂度表明算法在此种程度的基本操作中一定能完成工作。
  7. 平均时间复杂度是对算法的一个全面评价它完整全面的反映了这个算法的性质。但这种衡量并没有保证，不是每个计算都能在这个基本操作内完成，也会因为应用算法的实例分布可能并不均匀而难以计算。

因此，我们主要关注算法的最坏情况，亦即最坏时间复杂度。

时间复杂度的几条基本计算规则:

  1. 基本操作，即只有常数项，认为其时间复杂度为O(1)
  2. 顺序结构，时间复杂度按加法进行计算
  3. 循环结构，时间复杂度按乘法进行计算
  4. 分支结构，时间复杂度取最大值
  5. 判断一个算法的效率时，往往只需要关注操作数量的最高次项，其它次要项和常数项可以忽略
  6. 在没有特殊说明时，我们所分析的算法的时间复杂度都是指最坏时间复杂度

<6>数据结构

1.概念：

数据是一个抽象的概念，将其进行分类后得到程序设计语言中的基本类型。如：int，float，char等。数据元素之间不是独立的，存在特定的关系，这些关系便是结构。数据结构指数据对象中数据元素之间的关系。

2.算法与数据结构的区别

数据结构只是静态的描述了数据元素之间的关系。

高效的程序需要在数据结构的基础上设计和选择算法。

程序 = 数据结构 + 算法

总结：算法是为了解决实际问题而设计的，数据结构是算法需要处理的问题载体

关于正则表达式的一些总结

正则表达式

正则表达式：(Regular Expression)是计算机科学的一个概念。Regular Expression即“描述某种规则的表达式”之意。

正则表达式使用单个字符串来描述、匹配一系列匹配某个句法规则的字符串。在很多文本编辑器里，正则表达式通常被用来检索、替换那些匹配某个模式的文本。

1. 在Python中需要通过正则表达式对字符串进行匹配的时候通常引入：re模块

re.match是用来进行正则匹配检查的方法，若字符串匹配正则表达式，则match方法返回匹配对象（Match Object），否则返回None（注意不是空字符串””）。

匹配对象Macth Object具有group方法，用来返回字符串的匹配部分。

2. re模块示例(匹配以itcast开头的语句)

#coding=utf-8

import re

result = re.match("itcast","itcast.cn")

result.group()

运行结果为：

itcast

3. re.match() 能够匹配出以xxx开头的字符串。

4. 正则表达式中的表示字符

字符|代号

|||

.| 匹配任意1个字符（除了\n）

[ ] |匹配[ ]中列举的字符

\d |匹配数字，即0-9

\D | 匹配非数字，即不是数字

\s | 匹配空白，即 空格，tab键

\S | 匹配非空白

\w | 匹配单词字符，即a-z、A-Z、0-9、_

\W | 匹配非单词字符

需求：匹配出，一个字符串第一个字母为大小字符，后面都是小写字母并且这些小写字母可有可无

示例一：.

<<<re,match(".","m")
"m"
<<<

示例二·：[]

<<<re.match("[Hh]","hello world")#无论h大小写均可
"h"
<<<

5.Python中字符串前面加上 r 表示原生字符串

>>> ret = re.match(r"c:\\a",mm).group()
>>> print(ret)
c:\a

6.正则表达式中表示数量

字符| 功能

*|匹配前一个字符出现0次或者无限次，即可有可无

+|匹配前一个字符出现1次或者无限次，即至少有1次

?|匹配前一个字符出现1次或者0次，即要么有1次，要么没有

{m} |匹配前一个字符出现m次

{m,} |匹配前一个字符至少出现m次

{m,n} |匹配前一个字符出现从m到n次

示例一：*

>>>#coding =utf-8
>>>ret = re.match("[A-Z][a-z]*","Mm")
>>>ret.group()
"Mm"
>>>

>>>ret = re.match("[A-Z][a-z]*","Aabcdef")
>>>ret.group()
"Aabcdef"
>>>

示例二:?

需求：匹配出，0到99之间的数字

#coding=utf-8
import re

ret = re.match("[1-9]?[0-9]","7")
ret.group()

ret = re.match("[1-9]?[0-9]","33")
ret.group()

ret = re.match("[1-9]?[0-9]","09")
ret.group()

7.正则表达式表示边界

字符| 功能

^ |匹配字符串开头

$ |匹配字符串结尾

\b |匹配一个单词的边界

\B |匹配非单词边界

示例一：$

需求：匹配163.com的邮箱地址

8.正则表达式匹配分组

字符 |功能
|—-|——-|

| |匹配左右任意一个表达式

(ab) |将括号中字符作为一个分组

\num |引用分组num匹配到的字符串

(?P) |分组起别名

(?P=name) |引用别名为name分组匹配到的字符串

示例二:()

>>> ret = re.match("([^-]*)-(\d+)","010-12345678")
>>> ret.group()
'010-12345678'
>>> ret.group(1)
'010'
>>> ret.group(2)
'12345678'

9.re模块的高级用法

serch

需求：匹配出文章阅读的次数

#coding=utf-8
import re

ret = re.search(r"\d+", "阅读次数为 9999")
ret.group()

运行结果：

>>>ret.group()
"9999"
>>>

findall

需求：统计出python、c、c++相应文章阅读的次数

#coding=utf-8
import re

ret = re.findall(r"\d+", "python = 9999, c = 7890, c++ = 12345")
print ret

运行结果：

>>>print ret
["9999","7890","12345"]
>>>

sub :将匹配到的数据进行替换

需求：将匹配到的阅读次数加1

方法一：

#coding=utf-8
import re

ret = re.sub(r"\d+", '998', "python = 997")
print ret

运行结果:

>>>print ret
python = 998
>>>

方法二：

#coding=utf-8
import re

def add(temp):

10.贪婪和非贪婪

Python里数量词默认是贪婪的（在少数语言里也可能是默认非贪婪），总是尝试匹配尽可能多的字符；

非贪婪则相反，总是尝试匹配尽可能少的字符。

在”*”,”?”,”+”,”{m,n}”后面加上？，使贪婪变成非贪婪。

>>> s="This is a number 234-235-22-423"
>>> r=re.match(".+(\d+-\d+-\d+-\d+)",s)
>>> r.group(1)
'4-235-22-423'
>>> r=re.match(".+?(\d+-\d+-\d+-\d+)",s)
>>> r.group(1)
'234-235-22-423'
>>>

正则表达式模式中使用到通配字，那它在从左到右的顺序求值时，会尽量“抓取”满足匹配最长字符串，在上面的例子里面，“.+”会从字符串的启始处抓取满足模式的最长字符，其中包括想得到的第一个整型字段的中的大部分，“\d+”只需一位字符就可以匹配，所以它匹配了数字“4”，而“.+”则匹配了从字符串起始到这个第一位数字4之前的所有字符。

>>> re.match(r"aa(\d+)","aa2343ddd").group(1)
'2343'
>>> re.match(r"aa(\d+?)","aa2343ddd").group(1)
'2'
>>> re.match(r"aa(\d+)ddd","aa2343ddd").group(1) 
'2343'
>>> re.match(r"aa(\d+?)ddd","aa2343ddd").group(1)
'2343'
>>>

以上就是关于正则表达式基本知识，几乎所有的正则表达式都可以通过以上方法就行匹配，正则表达式广泛应用于检索，替换那些符合某个模式(规则)的文本，在python的爬虫应用中可以用来检索爬取检索到的有用信息。
